# プレゼンテーション作成スキル（slides-maker）

サントリー社内の標準スライドマスターに準拠したプレゼンテーションを作成する。python-pptx でテンプレートPPTXのマスターをそのまま使用し、ロゴ・アクセントバー・カラーが完全に再現される。日本語・英語の両言語に対応。

## トリガー条件

- プレゼンテーション・スライド・パワポ作成のリクエストで使用する
- 「プレゼン作って」「スライドを作って」「パワポ作成」「資料作成」等のリクエスト

## 前提条件

### python-pptx セットアップ

```bash
pip3 list 2>/dev/null | grep python-pptx || pip3 install python-pptx
```

### デザインシステムの読み込み

作業開始前に、必ず共通デザインシステム + 言語別リファレンスを読み込むこと:

```
Read ~/.claude/skills/slides-maker/references/design-system.md   # 共通（テンプレート、カラー、ヘルパー）
Read ~/.claude/skills/slides-maker/references/design-ja.md       # 日本語プレゼンの場合
Read ~/.claude/skills/slides-maker/references/design-en.md       # 英語プレゼンの場合
```

**言語の判定**: ユーザーの指示言語またはソース資料の言語に合わせる。明示的に指定がなければ日本語。

### テンプレートPPTX

スライドマスター（ロゴ・アクセントバー・カラー定義済み）:

```
~/.claude/skills/slides-maker/references/assets/template.pptx      # 日本語
~/.claude/skills/slides-maker/references/assets/template-en.pptx   # 英語
```

**テンプレートのスライドマスター0に4つのレイアウトが定義済み:**

| レイアウト | 用途 | 主要プレースホルダー |
|-----------|------|---------------------|
| `表紙` | タイトルスライド | idx=16: タイトル, idx=17: サブタイトル, idx=18: 部署, idx=11: 日付, idx=12: 作成者 |
| `アジェンダ` | 目次 | idx=10: コンテンツ, idx=4: ページ番号 |
| `本文` | コンテンツスライド | idx=15: **タイトル**, idx=10: **キーメッセージ**, idx=4: ページ番号 |
| `締め` | エンディング | なし（マスターに「以上」/「Thank You」テキスト済み、追加不要） |

## ワークフロー

### 1. 言語の確認

**最初にユーザーに言語を確認する:**

> 「プレゼンテーションは日本語と英語、どちらで作成しますか？」

- **日本語** → `design-ja.md` を読み込み、`template.pptx` を使用
- **英語** → `design-en.md` を読み込み、`template-en.pptx` を使用
- ユーザーが既に言語を明示している場合（例: 「英語でプレゼン作って」）はスキップしてよい

### 2. ユーザー資料の分析

アップロードされたファイルまたは提供されたテキストを調査し、以下を特定する:

- **キーメッセージ**: 各スライドで伝えるべき洞察・結論
- **データポイント**: 数値、パーセンテージ、トレンド、比較
- **適用可能なフレームワーク**: SWOT、2x2マトリクス、ウォーターフォール、ファネル、タイムライン
- **階層構造**: 全体的なナラティブアーク（物語構造）

### 3. プレゼンテーション計画をMarkdownで作成

**スライド構築の前に、必ずMarkdownで計画を作成しユーザーに提示すること。**
**計画はプレゼンテーションの言語に合わせて作成する（日本語→日本語計画、英語→英語計画）。**

#### 日本語の場合:

```markdown
# プレゼンテーション計画: [テーマ名]

## 全体構成

| # | レイアウト | タイトル | キーメッセージ |
|---|-----------|---------|---------------|
| 1 | 表紙 | メインタイトル | サブタイトル |
| 2 | アジェンダ | — | — |
| 3 | 本文 | AI活用の定義と目的 | AI活用は「効率化」と「意思決定の高度化」の2軸で推進 |
| ... | ... | ... | ... |
| N | 締め | — | — |

## 各スライド詳細

### Slide 3: AI活用の定義と目的
- **タイトル（idx=15）**: AI活用の定義と目的
- **キーメッセージ（idx=10）**: AI活用は「効率化」と「意思決定の高度化」の2軸で推進し、3プロジェクトが稼働中
- **コンテンツ構成**: 左カラム: 2軸の説明 / 右カラム: 3プロジェクトカード / 下部: KPI
- **ビジュアル**: 矩形カード、アクセントバー、KPI数値
```

#### 英語の場合:

```markdown
# Presentation Plan: [Theme]

## Structure

| # | Layout | Title | Key Message |
|---|--------|-------|-------------|
| 1 | Cover | Main Title | Subtitle |
| 2 | Agenda | — | — |
| 3 | Content | AI Strategy Overview | AI adoption focuses on two axes: efficiency and decision-making |
| ... | ... | ... | ... |
| N | End | — | — |

## Slide Details

### Slide 3: AI Strategy Overview
- **Title (idx=15)**: AI Strategy Overview
- **Key Message (idx=10)**: AI adoption focuses on two axes: operational efficiency and enhanced decision-making, with 3 projects currently active
- **Content layout**: Left column: two axes / Right column: 3 project cards / Bottom: KPIs
- **Visuals**: Rectangle cards, accent bars, KPI numbers
```

**計画のポイント:**
- タイトルはアジェンダレベルの**短いトピック名**
- キーメッセージはそのスライドの**洞察を述べる文**
- 各スライドのコンテンツ構成（カラム分割、テーブル、チャート等）を明記
- ユーザーの承認を得てからスライド構築に進む

### 4. 本文スライドの構造（重要）

本文スライドには3つのゾーンがある:

```
┌─────────────────────────────────────────┐
│ idx=15: タイトル（24pt, Bold）           │ 0.13" - 0.52"
│ アジェンダレベルの短いトピック名           │
├─────────────────────────────────────────┤
│ idx=10: キーメッセージ（16pt）            │ 0.81" - 1.51"
│ このスライドの洞察・結論を1文で            │
├─────────────────────────────────────────┤
│                                         │
│ 自由配置ゾーン（y=1.6" 以降）             │
│ テーブル、チャート、フレームワーク等       │
│                                         │
└─────────────────────────────────────────┘
```

- **タイトル（idx=15）**: アジェンダと対応する短いトピック名。24pt, Bold, `C_DARK`
- **キーメッセージ（idx=10）**: そのスライドの核心を伝える洞察文。16pt, `C_TEXT`。位置・サイズは明示指定（left=0.37", top=0.81", width=12.6", height=0.7"）
- **自由配置ゾーン**: y=1.6" 以降にテキストボックス・テーブル・シェイプで自由にレイアウト

### 5. デザインシステムに従ったスライド構築

`references/design-system.md` のデザインシステムに厳密に従うこと。主要原則:

**カラーパレット:**
- 背景: `FFFFFF`（ホワイト）
- プライマリテキスト: `333333`
- 見出し: `3E3A39`（ダークブラウン/グレー）
- チャートプライマリ: `00ABE4`（サントリーシアン）
- チャートシェード: `5BC2DC`, `91D7E7`, `B3E4EF`, `D9F0F7`
- アクセント: `FBAE40`（オレンジ、強調時のみ控えめに使用）
- グリッド/ボーダー: `CDCAC9`（ライトグレー）

**タイポグラフィ（言語別の詳細は `design-ja.md` / `design-en.md` を参照）:**

| 用途 | JA フォント | JA サイズ | EN フォント | EN サイズ |
|------|------------|----------|------------|----------|
| タイトル（idx=15） | Meiryo UI | 24pt Bold | Calibri | 24pt Bold |
| キーメッセージ（idx=10） | Meiryo UI | 16pt | Calibri | 16pt |
| アジェンダ項目 | Meiryo UI | 24pt | Calibri | 24pt |
| セクションヘッダー | Meiryo UI | 14-16pt Bold | Calibri | 14-16pt Bold |
| 本文 | Meiryo UI | 12-14pt | Calibri | 12-13pt |
| 大数値（KPI） | Calibri | 36-40pt Bold | Calibri | 36-40pt Bold |
| ソースライン | Meiryo UI | 10pt Italic | Calibri | 10pt Italic |

**レイアウトルール:**
- ロゴ・シアンアクセントバー・ページ番号枠はテンプレートマスターから自動適用（手動配置不要）
- 自由配置コンテンツは y=1.6" 以降に配置
- マルチカラムレイアウト推奨（2〜3カラム）
- セクション間にはバッファ（0.2-0.3"）を確保し、詰め込みすぎない

### 6. テーブルスタイリング

- ヘアラインボーダー（`CDCAC9`, 0.5pt）
- ヘッダー行はシアン（`00ABE4`）白テキスト
- 行の背景色を交互に（`F0F8FC` / `FFFFFF`）

### 7. フレームワークスライド

2x2マトリクス、比較フレームワーク、戦略ダイアグラムの場合:

- シェイプ（矩形＋ライン）で構築し、画像は使わない
- 矢印は `MSO_SHAPE.RIGHT_ARROW` 等のシェイプを使用（テキスト矢印「→」は不可）
- 細いライン（0.5-1pt）を`CDCAC9`で使用
- 各象限に薄いシアンティント背景
- 日本語テキストは必ずテキストボックスの幅を十分に確保（折り返し対策）

### 8. 言語別の注意点

**日本語（`design-ja.md`）:**
- フォント: Meiryo UI（見出し・本文）、Calibri（数値のみ）
- 日本語は英語より1-2pt大きめ設定。テキストボックスの幅も余裕を持たせる
- 全角句読点「。」「、」、「です・ます」調

**英語（`design-en.md`）:**
- フォント: Calibri 統一
- 英語は日本語より1-2pt小さめ（ラテン文字はコンパクトなため）
- Active voice、concise sentences。日付は "March 2026" 形式

### 9. 出力先の確認とスクリプト生成・実行

**最初にユーザーに出力先を確認する:**

> 「出力先ディレクトリはどこにしますか？（デフォルト: `/tmp`）」

- ユーザーが指定すればそのディレクトリを使用
- 指定がなければ `/tmp` に出力（全プロジェクトで確実に存在するため）

python-pptx スクリプトを `/tmp` に生成してPython3で実行する:

```bash
python3 /tmp/generate_slides.py
```

**出力ファイル名**: ユーザーが指定しなければ、テーマに基づいたファイル名を出力先ディレクトリに出力。

### 10. QAプロセス

生成後、以下を確認:

1. PPTXファイルが正常に生成されたか
2. ファイルサイズが妥当か（空でないか）
3. スライド枚数が計画通りか
4. エラーログがないか

## デザイン・アンチパターン（避けるべきこと）

- **タイトルに洞察文を入れる** — タイトル（idx=15）は短いトピック名、洞察はキーメッセージ（idx=10）に入れる
- **キーメッセージを空にして自由配置のみにする** — 必ずキーメッセージを活用する
- ビジュアル要素のない箇条書きだらけのスライド
- 3つ以上のフォントファミリーの混在
- ソース引用のないチャート
- ホワイトスペースのない詰め込みレイアウト（セクション間にバッファを確保）
- 定義済みパレット外の明るい・彩度の高い色の使用
- テキスト矢印（→、▼）の使用（MSO_SHAPEの矢印シェイプを使う）
- ネイビー/ブルー系のコンサルティングカラーの使用（サントリーシアン系を使うこと）
- Yu Mincho（明朝体）の使用（Meiryo UIを使うこと）
- ロゴやアクセントバーの手動配置（テンプレートマスターから自動適用されるため不要）
- 締めスライドへのテキスト追加（マスターに「以上」/「Thank You」テキスト済み）
